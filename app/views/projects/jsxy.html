CoffeeScript-JSXY

CoffeeScript-JSXY adds Haml-inspired whitespace-indented JSX syntax to CoffeeScript 2

Here we’ll illustrate the new JSXY syntax. For other info like installation/background/breaking changes, see the coffeescript-jsxy README. See the CoffeeScript 2 docs for CoffeeScript usage and syntax

JSXY

JSXY aims to provide a nicer, whitespace-indented JSX syntax that fits well into CoffeeScript’s existing whitespace-indented style. Its syntax is heavily inspired by Haml. Standard JSX syntax is also supported, but here we’ll look at the whitespace-indented syntax (compiled JS output on right)

Elements

Create JSX elements with %element. An import React from react will automatically be included at the top of the compiled JavaScript module. It’s your responsibility to include react in your project.

%div

import React from 'react';

<div></div>;

Attributes/props can be supplied using {} or (). {} attributes are parsed as CoffeeScript objects so you can use eg object literal shorthands.

%input{
  type: 'text'
  @onChange
  props...
}

%Greeting( name='Juan' )

import React from 'react';

<input type={'text'} onChange={this.onChange} {...props}></input>;

<Greeting name='Juan'></Greeting>;

You can use Haml-style #id and .class shorthands.

#app

.container

%h1.heading#main_title

.col-sm-6.col-md-3

import React from 'react';

<div id='app'></div>;

<div className='container'></div>;

<h1 id='main_title' className='heading'></h1>;

<div className='col-sm-6 col-md-3'></div>;

With a .class element that’s directly preceded by CoffeeScript code, be careful to leave a blank line to distinguish it from a chained method call.

a

.b(x='c')

a
.b(x='c')

var x;

import React from 'react';

a;

<div className='b' x='c'></div>;

a.b(x = 'c');

Since JSX allows element names like Form.Field, the second half is interpreted as part of the element name if it’s capitalized.

%Form.Field

%Form.field

import React from 'react';

<Form.Field></Form.Field>;

<Form className='field'></Form>;

If you want to use a capitalized class name, use dynamic class name syntax (see below) or a non-shorthand className attribute.

%div.('Class1')

%div{ className: 'Class1' }

import classNames from 'classnames';

import React from 'react';

<div className={classNames('Class1')}></div>;

<div className={'Class1'}></div>;

If you want to use a non-capitalized dotted component name, dereference it first.

{field: Field} = Form

%Field

var Field;

import React from 'react';

({
  field: Field
} = Form);

<Field></Field>;

For dynamic class names, use .() shorthand. This will compile to a call to classNames(), so picture the parentheses as surrounding the arguments to classNames(). It will also automatically include an import classNames from classnames at the top of the compiled JavaScript module. It’s your responsibility to include classnames in your project.

%section.(
  highlighted: index > 2
  'active active-2': @props.is_active
)

import classNames from 'classnames';

import React from 'react';

<section className={classNames({
  highlighted: index > 2,
  'active active-2': this.props.is_active
})}></section>;

Nest child elements using indentation.

.row
  .col-sm-3
    %h1( title='Accounts' ) Accounts
    %Accounts{ accounts: @props.all_accounts }

import React from 'react';

<div className='row'>
  <div className='col-sm-3'>
    <h1 title='Accounts'>
      Accounts
    </h1>
    <Accounts accounts={this.props.all_accounts}></Accounts>
  </div>
</div>;

Text content

Text content can be indented or inline.

%h1 Greetings

%p
  Welcome to Top Notch Hotels.
  We hope you enjoy your stay

import React from 'react';

<h1>
  Greetings
</h1>;

<p>
  Welcome to Top Notch Hotels.
  We hope you enjoy your stay
</p>;

Expressions

To include expressions, use JSX-style {} or Haml-style =.

%h1= page_title

%h2
  Welcome, {name}!

import React from 'react';

<h1>
  {page_title}
</h1>;

<h2>
  Welcome, {name}!
</h2>;

Since (in React) expressions/elements on adjacent lines won’t be separated by any whitespace by default, there’s a ^ operator to force an expression or element to include a surrounding space between itself and preceding/following siblings.

%p
  Oh man,
  =^ name
  is going to
  %strong{ style: fontSize: '2em' }^
    = action
  when he finds out!

import React from 'react';

<p>
  Oh man, {name} is going to <strong style={{
    fontSize: '2em'
  }}>
    {action}
  </strong> when he finds out!
</p>;

Since in CoffeeScript “everything is an expression”, you get some much nicer/more natural ways of doing things than in straight JSX/ES6. To conditionally render a subcomponent, use if.

%div
  = if headerText
    %Header{ headerText }

import React from 'react';

<div>
  {(headerText ? <Header headerText={headerText}></Header> : void 0)}
</div>;

You can use any of the CoffeeScript conditional syntaxes, like unless.

%div
  = unless dontIncludeHeader
    %Header{ headerText }

import React from 'react';

<div>
  {(!dontIncludeHeader ? <Header headerText={headerText}></Header> : void 0)}
</div>;

Or inline postfix if/unless.

%div
  = %Header{ headerText } if headerText?
  = %Footer unless dontIncludeFooter

import React from 'react';

<div>
  {(typeof headerText !== "undefined" && headerText !== null ? <Header headerText={headerText}></Header> : void 0)}
  {(!dontIncludeFooter ? <Footer></Footer> : void 0)}
</div>;

For alternatives, use else/else if. As of now, when using = syntax, you must indent your else/else if lines relative to the initial = if line.

%div
  = if loggedIn
    %Welcome
   else if forbidden
    %GoAway
   else
    %Login

import React from 'react';

<div>
  {(loggedIn ? <Welcome></Welcome> : forbidden ? <GoAway></GoAway> : <Login></Login>)}
</div>;

Or use CoffeeScript’s very nice switch syntax.

Home = ({loggedIn, forbidden}) ->
  %div
    = switch
      when loggedIn  then %Welcome
      when forbidden then %GoAway
      else                %Login

var Home;

import React from 'react';

Home = function({loggedIn, forbidden}) {
  return <div>
    {((function() {
      switch (false) {
        case !loggedIn:
          return <Welcome></Welcome>;
        case !forbidden:
          return <GoAway></GoAway>;
        default:
          return <Login></Login>;
      }
    })())}
  </div>;
};

To loop over child components, you can use for (rather than having to use .map(callback)).

render: ->
  {steps} = @props

  %ul.steps-container
    = for {ingredients, description}, i in steps
      %Step{
        ingredients, description
        key: i
      }

import React from 'react';

({
  render: function() {
    var description, i, ingredients, steps;
    ({steps} = this.props);
    return <ul className='steps-container'>
      {(function() {
        var j, len, results;
        results = [];
        for (i = j = 0, len = steps.length; j < len; i = ++j) {
          ({ingredients, description} = steps[i]);
          results.push(<Step ingredients={ingredients} description={description} key={i}></Step>);
        }
        return results;
      })()}
    </ul>;
  }
});

Or use inline postfix for.

render: ->
  {steps} = @props

  %ul.steps-container
    = %Step{
      ingredients, description
      key: i
    } for {ingredients, description}, i in steps

import React from 'react';

({
  render: function() {
    var description, i, ingredients, steps;
    ({steps} = this.props);
    return <ul className='steps-container'>
      {(function() {
        var j, len, results;
        results = [];
        for (i = j = 0, len = steps.length; j < len; i = ++j) {
          ({ingredients, description} = steps[i]);
          results.push(<Step ingredients={ingredients} description={description} key={i}></Step>);
        }
        return results;
      })()}
    </ul>;
  }
});

Angular-style | filters are also supported inside JSXY expressions. Since : is heavily used in CoffeeScript, the additional-argument delimiter is ~. You don’t have to specially register filters like in Angular, any function can be used as a filter.

%a ${amount | currency}

%strong= octalVal | parseInt ~ 8

import React from 'react';

<a>
  ${currency(amount)}
</a>;

<strong>
  {parseInt(octalVal, 8)}
</strong>;
